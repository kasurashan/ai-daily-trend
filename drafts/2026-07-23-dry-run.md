## Slack 게시 항목

### 1. OpenAI API hard spend limits: 월간 cap 도달 시 API 요청이 `429`로 차단됨

- 상태: `new`
- 분류: `headline`
- 핵심 메시지: OpenAI API changelog는 2026-07-22에 organization 및 project 단위 hard spend limits를 추가했다고 공지했다. 설정한 월간 cap에 도달하면 해당 API 요청이 `429` 오류를 반환한다.
- 실제 새로움: 기존 spend alerts가 사전 경고에 가까웠다면, hard spend limits는 비용 초과 시 traffic interruption을 의도적으로 발생시키는 운영 제어 장치다.
- 구체적 기술 디테일: production inference client는 `429`를 단순 rate limit로만 보지 말고 budget cap 도달 신호로 분기해야 한다.
- 기존 대비 달라진 점: 비용 알림 중심에서 요청 차단까지 포함하는 budget guardrail로 바뀐다.
- 실무적 의미: 월말 batch, agent loop, high-volume evaluation job이 예산 cap에 걸리면 정상 rate-limit와 같은 코드 경로로 실패할 수 있다.
- Takeaway: production client와 batch runner의 `429` handling을 점검하고, spend cap 도달 시 fallback, queue pause, owner alert를 분리해야 한다.
- 검증 상태: 공식 OpenAI API changelog 기준.
- 주요 한계: 실제 cap 설정 UI, project별 propagation 지연, 기존 rate-limit `429`와의 식별 방법은 운영 환경에서 확인해야 한다.
- 원출처: [OpenAI API changelog](https://developers.openai.com/api/docs/changelog)
- 보조 출처: 없음

### 2. PyTorch 2.13: FlexAttention Apple Silicon 지원과 `nn.LinearCrossEntropyLoss` memory 절감

- 상태: `new`
- 분류: `headline`
- 핵심 메시지: PyTorch Foundation은 2026-07-23 업데이트에서 PyTorch 2.13의 주요 변화로 Apple Silicon FlexAttention, CuTeDSL Inductor backend, `nn.LinearCrossEntropyLoss`, Python 3.15 wheel 지원을 정리했다.
- 실제 새로움: Apple Silicon에서 FlexAttention이 SDPA 대비 최대 약 12배 빠르다고 보고됐고, `nn.LinearCrossEntropyLoss`는 peak GPU memory를 최대 약 4배 줄이는 방향의 API다.
- 구체적 기술 디테일: attention kernel 선택과 loss 계산 경로가 각각 local inference 또는 training memory 병목에 직접 영향을 준다.
- 기존 대비 달라진 점: PyTorch 2.13은 단순 platform support보다 compiler/backend, attention, loss memory 경로의 변경이 같이 들어간 release로 봐야 한다.
- 실무적 의미: Mac 기반 실험 환경이나 memory-bound training recipe를 쓰는 팀은 작은 코드 변경으로 병목이 달라질 수 있다.
- Takeaway: Apple Silicon local eval과 memory-bound fine-tuning job은 PyTorch 2.13에서 동일 batch, dtype, sequence length 조건으로 다시 benchmark해야 한다.
- 검증 상태: 공식 PyTorch Foundation 업데이트 및 release blog 기준.
- 주요 한계: 최대 성능 수치는 workload 조건 의존적이며, Python 3.15 지원은 torch wheel 중심이고 `torch.compile` 지원 제약이 있다.
- 원출처: [PyTorch Foundation update](https://pytorch.org/blog/driving-the-future-of-open-source-ai-an-update-from-pytorch-foundation-projects/)
- 보조 출처: [PyTorch 2.13 release blog](https://pytorch.org/blog/pytorch-2-13-release-blog/)
